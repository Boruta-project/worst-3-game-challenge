## ğŸ”¬ Assumptions & design notes

- ğŸ—‚ï¸ **One file to rule them all.** The App Builder tends to keep everything in
  `page.tsx`. This is not best practice. It is, however, very funny.
- ğŸ¯ **`currentGame === 'tetris'` controls the Space Shooter.** The third game
  changed from Tetris to a space shooter during development, but the internal
  type was never renamed. A feature, not a bug.
- ğŸ“ **The maze is 32 columns wide.** Except when it isn't â€” two rows shipped
  33 characters wide, shifting the right border one column to the right.
  Fixing that required counting characters by hand like it was 1994.
- ğŸ‘¾ **Enemy spawn intervals.** The AI put `shooterEnemies` in a `useEffect`
  dependency array, which caused the spawn timer to reset every 150 ms and
  never fire. Enemies only appeared at the start of the game. Human review
  found this; a follow-up prompt fixed it with `useRef`.
- ğŸ¤” **Vibecoding has limits.** The models are excellent at generating large
  amounts of coherent code quickly. They are less reliable at maintaining
  consistency across 1 500 lines in a single file. Human review and targeted
  correction prompts remain necessary â€” especially for React hooks semantics.
